# Word 格式工具角色与视觉安全增强设计

## 背景

真实 Agent 使用暴露出两个高风险问题：

1. 无样式的中英文论文题目可能被识别为 `body`，随后被套用正文格式。
2. 页面尺寸或边距修改会触发 Word 重新排版，现有内容指纹无法发现封面、
   浮动对象和分页的视觉损坏。

同时，单一 `title` 和 `reference_entry` 角色无法表达中英文不同格式。

## 目标

- 可靠识别封面区域中的中文、英文论文题目。
- 对无法可靠识别的封面段落默认不执行正文格式修复。
- 支持中文/英文题目和中文/英文参考文献的差异规则。
- 页面几何修改前检测封面表格、浮动对象、文本框和 VML 图形。
- 默认阻止高风险页面几何自动修复，允许规则显式覆盖。
- 内容完整性校验增加浮动对象 OOXML 位置指纹。
- 提供可选的 DOCX 渲染与 PDF 页面视觉比较，不作为核心修复的强依赖。
- 保持旧 `title`、`reference_entry` 和 `reference` 规则可用。

## 角色识别

新增角色：

- `title_zh`
- `title_en`
- `reference_entry_zh`
- `reference_entry_en`

旧角色仍保留。规则回退顺序：

- `title_zh` / `title_en` -> `title`
- `reference_entry_zh` / `reference_entry_en` -> `reference_entry` -> `reference`
- `reference_heading` -> `heading_1`

标题识别按以下优先级执行：

1. `title_zh_patterns` / `title_en_patterns` 显式规则。
2. 旧 `title_patterns` 或 Word Title 样式，再按字符语言拆分。
3. 封面上下文启发式：
   - 位于目录、摘要或正文首章之前；
   - 中文标题包含足够中文字符，字号较大，通常居中；
   - 英文标题以拉丁字符为主，可与已识别中文标题相邻；
   - 排除学校名称、论文类型、作者、学院、专业、学号、指导教师等元数据。

当检测到标题和明确的封面结束标记时，其余未识别封面段落标记为
`protected_front_matter`，不套用正文格式。

参考文献条目根据编号之后的主要语言拆分；混合或无法判断的条目继续使用
`reference_entry`。

## 页面几何安全

新增布局风险分析，记录：

- `wp:anchor` 浮动 Drawing；
- `w:txbxContent` / VML textbox；
- VML shape / pict；
- 正文开始前的封面表格；
- Word 分节数量。

若存在页面几何偏差且命中高风险对象：

- 检查报告保留边距或页面尺寸问题；
- 相关问题默认 `fixable=false`；
- 新增 `page_layout_risk` 说明阻断原因；
- `fix` 不修改这些页面字段。

规则可通过 `priority.allow_unsafe_page_geometry=true` 显式覆盖。该覆盖只表示调用方
接受风险，不代表视觉结果已经验证。

## 完整性与视觉验证

内容指纹新增浮动和内联对象的定位、尺寸、环绕及关联标识。修复输出若改变原有
对象定位 OOXML，原子发布失败。

新增可选 `visual-compare` 流程：

1. 在 Windows 上通过 Microsoft Word 将源 DOCX 和候选 DOCX 只读导出为 PDF。
2. 使用可选视觉依赖渲染 PDF 页面。
3. 比较页数、页面尺寸和逐页像素变化比例。
4. 输出 JSON 报告及可选差异图。

视觉比较只报告风险，不自动判定论文格式是否正确。缺少 Word 或视觉依赖时返回
明确的可用性错误，不影响普通检查和修复。

## 模板抽取

模板 profile 继续提供事实，不直接自动生成规则。新增：

- 上下文角色识别结果；
- `protected_front_matter` 标记；
- 布局风险摘要；
- 提醒 Agent 对低确定性角色和页面几何进行人工决策。

## 扩展文档区域

分析器增加对以下区域的清单与格式处理：

- 表格单元格，包括封面表格；
- 分节页眉；
- 分节页脚。

每个分析段落带稳定的区域和位置描述，正文段落索引保持兼容。封面表格使用保守的
`cover_label`、`cover_value` 角色，其他表格使用 `table_text`；页眉页脚分别使用
`header`、`footer`。这些区域只有存在显式角色规则时才修复，绝不回退到 `body`。

文本框继续只报告不修复，因为 `python-docx` 对 DrawingML 和旧 VML 文本框没有统一、
稳定的段落对象接口。

## 章节结构与目录

新增致谢和附录角色，并为模板 profile 输出章节结构清单。结构修改与普通 `fix`
分离：

1. `plan-structure` 对比模板与初稿，报告缺失章节和不支持复制的块。
2. 显式结构命令只补建规则声明的缺失章节标题或占位段落。
3. 不复制任意模板正文，不替换初稿已有章节。

目录支持分三层：

1. 使用显式 `table_of_contents` 规则修复目录段落样式。
2. 仅在目录可见标题能唯一匹配已识别正文标题时修复缺失 `PAGEREF` 目标并补建书签。
3. 设置 Word 的 `updateFields`，使输出文件打开时刷新目录、页码和其他域。

无法唯一匹配的目录条目保持不可修复并进入未覆盖清单。完整分页仍由 Word 或兼容
渲染器完成，工具不伪造页码结果。

## 覆盖范围报告

检查和修复报告增加机器可读的覆盖摘要：

- 已检查区域和角色数量；
- 受保护封面段落；
- 不支持的文本框和浮动对象；
- 无法确定性修复的目录条目；
- 缺失结构角色；
- 是否执行视觉比较。

普通 `fix` 的输出称为“支持范围内的格式修复”，只有覆盖检查满足时才允许上层
Agent 将其描述为完整套版。

## 测试

- 无样式真实形态的中英文标题识别。
- 英文标题非居中但紧随中文标题时仍可识别。
- 封面说明和身份信息不会被识别为标题或正文。
- 旧 `title` 规则兼容。
- 中英文参考文献应用不同对齐规则。
- 浮动对象、文本框或封面表格阻止页面几何自动修复。
- 显式风险覆盖允许页面几何修复。
- 浮动对象位置指纹变化会阻止发布。
- 表格封面、页眉和页脚仅按显式规则修复。
- 目录书签只在唯一匹配时重建，并设置打开时更新域。
- 报告列出文本框、受保护封面和不可修复目录等未覆盖区域。
- 视觉比较的页数、页面尺寸和像素差异报告。
- 既有全部测试和真实文档流程回归。
